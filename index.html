COMPONENT RULES

组件规范 · 状态规则

Buttons

Primary 高 38-48px，圆角 8px；Hover 加深，Active 压低，Disabled 使用 fog 背景和浅色文字。

Chips

高度 28-30px，圆角 full，字体 12px / 600；按语义使用红、紫、绿、黄、灰浅底。

Cards

白底、1px 边框、8-16px 圆角、轻阴影；课程卡必须有封面、标题、meta、进度。

Alerts

Success 绿、Warning 黄、Error 红、Info 蓝或紫；图标和说明文字必须清楚区分。

Avatar

AI 助手入口用牛小匠 standard；系统提示用 thinking；完成反馈用 celebration。

Progress

学习进度优先用紫色 fill；步骤条明确 done / active / waiting 三种状态。

Empty States

使用低密度布局、简短文案、明确下一步 CTA，可搭配小尺寸牛小匠。

Navigation

移动端 5 项以内，桌面侧边栏强调当前状态；标签、筛选、分页必须能同步 URL。

08

DO / DON'T

设计红线 · 品牌隔离

Do · 必须做

  • 主 CTA 使用黑色或品牌红，保持克制。
  • 页面大背景使用白色、浅灰或米白。
  • 牛小匠保持红 JR 连帽衫、墨镜、JR 胸标。
  • 学习进度使用紫色，完成状态使用绿色。
  • Card 圆角 8-16px，阴影轻，不做厚重拟物。
  • 文案用「你」，直接说用户下一步要做什么。

Don't · 禁止

  • 不要大面积铺满红色背景。
  • 不要用蓝色作为 JR Academy 主 CTA。
  • 不要去掉牛小匠的墨镜、红衣、JR 胸标。
  • 不要混用 UniMate / Job Hunter / Cert Master 的 mascot outfit。
  • 不要在一屏重复堆很多牛小匠。
  • 不要写空洞模板话、过度营销话。

DECORATIONS

装饰元素 · 跨产品复用规则

资产库

  • Shapes：Sparkle 4/5-pt · Dot · Ring · Soft Blob。
  • Arrows：Straight · Curved · Scribble。
  • Highlights：Highlighter · Wave Underline · Hand-Drawn Circle · Dashed Divider。
  • Accents：Bubble Quote · Number Badge。

用色规则

  • Sparkle / Highlighter / Wave → 黄 #FBBF24
  • Arrow / Hand Circle / 强情绪 → 红 #EF4444
  • Scribble / Ring → 紫 #6366F1
  • Dot / Bubble 中性 → 灰阶

使用决策树

  • 关键词高亮：第一选 Highlighter，第二选 Wave Underline。
  • 圈住一段：用 Hand-Drawn Circle，不用 box-shadow。
  • 引导视线：Arrow Straight（CTA 内）；Arrow Curved（跨段连接）。
  • hero 灵动感：散落组合 Sparkle + Dot + Ring（见 panel 18）。

Do · 必须做

  • 装饰只出现在 hero / 顶部容器 / 庆祝场景，不进入数据视图。
  • 颜色取自 brand token（红 / 黄 / 紫 / 中性灰），不引入新色。
  • SVG 用 currentColor，靠外层 class 控色。
  • 一屏装饰元素 6-10 个为宜，过密 = 廉价。

Don't · 禁止

  • 不要在 Dashboard / 数据表格 / 长正文里散落 sparkle。
  • 不要把 Arrow 用作 CTA 主图标（CTA 自己有 → 字符即可）。
  • 不要同时叠 Highlighter + Wave Underline + Hand Circle（视觉过载）。
  • 不要给装饰加大阴影 / 厚重描边——保持轻盈。
React 项目（jr-academy-web-zh）建议在 src/components/decor/ 实现一套 React 版，SVG path 必须跟本手册 panel 16-18 完全一致，不要二次创作。Chrome 插件 / 小程序 / 静态页直接 copy 本手册的 SVG 内联即可。
