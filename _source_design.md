# JR Academy Design System — 牛小匠版

> 版本：v1.0  
> 适用范围：JR Academy 官网、学习平台、AI 职业诊断、移动端页面、课程卡片、运营物料  
> 核心调整：将原有吉祥物统一替换为 **牛小匠**，视觉风格参考最新 AI 职业诊断页面中的红色连帽衫小牛形象。

---

## 1. 品牌定位

JR Academy 的视觉系统需要传达三个关键词：

1. **AI 时代感**：面向 AI 学习、职业成长、技能升级。
2. **可信赖的教育品牌**：界面清晰、结构专业、学习路径明确。
3. **年轻、有陪伴感**：通过牛小匠降低学习压力，增强陪伴感和品牌记忆点。

品牌口号建议继续使用：

**开启盒子，开启未来**  
**OPEN THE BOX. OPEN YOUR FUTURE**

在 AI 职业诊断、学习路径、课程推荐等场景中，可以增加更具成长感的表达：

**学好 AI，未来更牛！**  
**Smarter Study, Better Future!**

## 2. 吉祥物系统：牛小匠

### 2.1 角色定义

**名称：牛小匠**  
**角色身份：JR Academy AI 学习伙伴 / 职业成长陪伴官**

牛小匠是一个可爱、聪明、有行动力的小牛角色，代表 JR Academy 学员在 AI 时代不断学习、探索方向、提升能力、链接机会的成长状态。

### 2.2 视觉特征

牛小匠需要保持以下统一特征：

- 黑白小牛 / 小匠人形象
- 小牛角，圆润可爱，不能过于凶猛
- 黑色头发或牛头轮廓，整体偏圆
- 戴黑色墨镜，增强酷感和记忆点
- 穿红色 JR 连帽衫或红色运动套装
- 胸前保留 `JR` 标识
- 表情积极、自信、友好
- 线条圆润，适合 UI、贴纸、头像、课程卡片、小尺寸图标使用

### 2.3 风格方向

牛小匠应参考最后一张 AI 职业诊断视觉稿中的风格：

- 更偏 **柔和、轻量、现代插画感**
- 不要过度厚重或 3D 化
- 表情更亲和，不要过于“黑酷”
- 红色衣服可以保留品牌识别，但整体颜色应更干净
- 可用于网页 hero、移动端引导页、学习提示、空状态、成就反馈等场景

### 2.4 替换原则

原视觉稿中的旧吉祥物应全部替换为牛小匠，包括：

- Web Dashboard 右上角头像
- Web Dashboard 左侧底部引导卡
- 推荐课程卡片中的吉祥物插画
- Mobile 首页 hero 区吉祥物
- Mobile 学习页课程卡片封面
- Components 中 Avatar 区域
- Mascot IP 区域所有正面、背面、侧面、学习、讲解、庆祝等动作
- Toast、Alert、空状态中需要出现的吉祥物提示形象

替换时不要改变原 UI 结构，只替换角色形象和相关命名。

## 3. 色彩系统

### 3.1 Brand Colors 品牌色

| 名称 | 色值 | 用途 |
|---|---:|---|
| Coral Red | `#FF4D4F` | 主按钮、重点标签、品牌强调、CTA |
| Royal Purple | `#7861FF` | 学习进度、AI 模块、图表强调 |
| Sunglow Yellow | `#FFC94D` | 成就、提醒、机会、成长感 |
| Mint Green | `#10B981` | 成功状态、完成状态、正向反馈 |
| Ink Black | `#111111` | 主文字、Logo、重要标题 |
| Mist Gray | `#F5F6F7` | 页面背景、卡片背景、分割区域 |

### 3.2 Neutral Colors 中性色

| 色值 | 用途 |
|---:|---|
| `#1F1F1F` | 一级标题、重点文字 |
| `#333333` | 正文标题、导航文字 |
| `#666666` | 次级说明文字 |
| `#999999` | 辅助信息、弱提示 |
| `#E5E7EB` | 分割线、边框 |
| `#F5F6F7` | 浅灰背景、输入框背景 |

### 3.3 使用规则

- 主 CTA 使用 `#FF4D4F`。
- 学习进度条优先使用 `#7861FF`。
- 成功状态使用 `#10B981`。
- 警告和成长激励可以使用 `#FFC94D`。
- 大面积背景保持白色或浅灰，避免影响教育产品的专业感。
- 牛小匠服装以红色为主，不建议随意改变主服装颜色。

## 4. 字体系统

### 4.1 英文字体

推荐使用：

**Plus Jakarta Sans**

适用于：

- 英文标题
- 英文标签
- 数据指标
- Dashboard 信息结构

### 4.2 中文字体

推荐使用：

**思源黑体 / Source Han Sans**

适用于：

- 中文标题
- 正文说明
- 按钮文字
- 移动端 UI

### 4.3 字号规范

| 层级 | 字号 | 字重 | 用途 |
|---|---:|---|---|
| H1 | 32px | Bold | 大标题、Hero 主标题 |
| H2 | 24px | Bold | 页面区块标题 |
| H3 | 20px | Semibold | 卡片标题、模块标题 |
| Body-1 | 16px | Regular | 正文内容 |
| Body-2 | 14px | Regular | 课程说明、列表文本 |
| Caption | 12px | Medium | 辅助信息、标签、数据说明 |

1. **左侧导航栏**

```text
开启盒子
```


- 社区交流

左下角引导卡建议使用：

## 7. 组件系统

### 7.1 Buttons 按钮

#### Primary Button

用途：主要 CTA，例如开始诊断、继续学习、立即探索。

样式：

- 背景：`#FF4D4F`
- 文字：白色
- 圆角：8px - 12px
- Hover：略微加深
- Active：降低亮度或增加阴影压感

示例：

```text
开始诊断 →
继续学习 →
立即探索 →
```

#### Secondary Button

用途：次级操作，例如查看详情、去测评、查看路径。

样式：

- 白色背景
- 红色描边或浅灰描边
- 文字使用品牌红或深灰

### 7.2 Chips 标签

常用标签：

- AI 助手
- Lecture
- Quiz
- Assignment
- 重要
- 进行中
- 已完成
- 草稿
- 自定义

状态颜色：

| 状态 | 建议颜色 |
|---|---|
| 重要 | 浅红背景 + 红色文字 |
| 进行中 | 浅紫背景 + 紫色文字 |
| 已完成 | 浅绿背景 + 绿色文字 |
| 草稿 | 浅灰背景 + 灰色文字 |

### 7.3 Cards 卡片

卡片统一规范：

- 背景：白色
- 圆角：16px
- 边框：`#E5E7EB`
- 阴影：轻量
- 内边距：16px - 24px
- 图片区域可使用渐变背景

课程卡片必须包含：

- 课程封面
- 课程标题
- 学习人数
- 学习进度
- 进度条

### 7.4 Avatar 头像

头像系统可包含：

- 牛小匠标准头像
- 牛小匠墨镜头像
- 学员头像
- 导师头像

牛小匠头像适合用于：

- AI 助手入口
- 系统提示
- 学习陪伴
- 消息通知
- 空状态

## 8. Alert & Toast 规范

### 8.1 Alerts 提示组件

| 类型 | 颜色 | 示例文案 |
|---|---|---|
| Success | 绿色 | 操作成功！你的学习计划已成功创建。 |
| Warning | 黄色 | 注意：请完善你的个人资料以获得更准确的推荐。 |
| Error | 红色 | 操作失败：网络连接超时，请重试。 |
| Info | 紫色 | 提示：AI 正在为你生成学习路径，请稍候... |

### 8.2 Toast 轻提示

常用 Toast：

```text
学习打卡成功！
已加入学习计划
删除成功
```

Toast 可以搭配牛小匠小头像，用于提升亲和力，但不要在高频操作中滥用。

## 9. 插画系统

### 9.1 插画主题

当前设计系统中建议保留四类插画场景：

1. **探索方向**
2. **提升能力**
3. **链接机会**
4. **成长未来**

### 9.2 插画风格

插画需要与牛小匠保持统一：

- 圆角卡片
- 柔和渐变
- 轻量人物或抽象场景
- 使用红、紫、黄、绿作为主辅助色
- 不要使用过重阴影或复杂背景

## 10. Mascot IP 区域规范

### 10.1 必备视图

牛小匠 IP 资产至少应包含：

- 正面 Front
- 侧面 Side
- 背面 Back
- 坐姿 Sitting
- 拿电脑 Laptop
- 拿平板 Tablet
- 挥手 Wave
- 点赞 Like
- 思考 Thinking
- 讲解 Teaching
- 庆祝 Celebration

### 10.2 表情与动作

建议优先制作以下动作，用于产品和运营高频场景：

| 动作 | 使用场景 |
|---|---|
| 开心 | 任务完成、学习成功 |
| 思考 | AI 诊断、问题引导 |
| 点赞 | 完成课程、获得成就 |
| 讲解 | 课程页、AI 助手说明 |
| 努力 | 学习中、进度提醒 |
| 庆祝 | 证书、徽章、活动成功 |
| 疑问 | FAQ、错误输入、引导解释 |
| 加油 | 学习计划、打卡、提醒 |

### 10.3 禁止事项

- 不要把牛小匠画得过于写实。
- 不要去掉红色 JR 服装识别。
- 不要使用过于攻击性或严肃的牛角形态。
- 不要让墨镜遮挡导致角色表情完全不可识别。
- 不要混用旧吉祥物与牛小匠。
- 不要在同一页面中使用两套不同风格的牛小匠。

## 14. 总结

本次设计调整的核心不是简单替换图片，而是将 JR Academy 的视觉系统从“普通教育平台 UI”升级为拥有明确 IP 记忆点的 **AI 学习陪伴型品牌系统**。

牛小匠需要承担三个角色：

1. **品牌识别角色**：让用户一眼记住 JR Academy。
2. **学习陪伴角色**：在诊断、课程、任务、反馈中陪伴用户。
3. **成长激励角色**：通过可爱、积极、有行动力的形象鼓励用户持续学习。

最终目标是让 JR Academy 在 AI 教育和职业成长产品中形成更强的视觉辨识度和情感连接。
